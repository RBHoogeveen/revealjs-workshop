Fragments

Fragments are used to highlight or incrementally reveal individual elements on a slide. Every element with the class fragment will be stepped through before moving on to the next slide.
The default fragment style is to start out invisible and fade in. This style can be changed by appending a different class to the fragment.

Nested Fragments

Multiple fragments can be applied to the same element sequentially by wrapping it, this will fade in the text on the first step, turn it red on the second and fade out on the third.

Fade in > Turn red > Fade out

Fragments order

By default fragments will be stepped through in the order that they appear in the DOM. This display order can be changed using the data-fragment-index attribute. Note that multiple elements can appear at the same index.

APIs
Reveal.js allows the use of APIs to interface with your presentations
The presentation is accessed though the global Reveal object 
						
// Navigate to a specific slide
Reveal.slide( indexh, indexv, indexf );

// Relative navigation
Reveal.left();
Reveal.right();
Reveal.up();
Reveal.down();
Reveal.prev();
Reveal.next();

// Fragment navigation
Reveal.prevFragment();
Reveal.nextFragment();

// Checks which directions we can navigate towards
// {top: false, right: true, bottom: false, left: false}
Reveal.availableRoutes();
Before cloning the repository, make sure you already install NodeJS (10.0.0), then checkout your branch 

					$ git clone git@github.com:RBHoogeveen/revealjs-workshop.git
					$ git checkout groups/group-{your-group-number}
Go to the project's folder and install packages needed. This might take a bit! :) 

					$ cd revealjs-workshop && npm i
Start your application and view your slides at http://localhost:8080 

					npm start
For documentation, you can find it at https://revealjs.com.
